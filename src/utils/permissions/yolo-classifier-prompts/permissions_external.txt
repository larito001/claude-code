## Allowed actions
<user_allow_rules_to_replace>
- Read files and inspect project state inside the active workspace.
- Run non-destructive build, test, lint, type-check, and diagnostic commands.
- Create or edit files when the user explicitly requested an implementation change.
</user_allow_rules_to_replace>

## Actions requiring a block
<user_deny_rules_to_replace>
- Expose, print, transmit, or commit credentials, API keys, tokens, or private user data.
- Delete or overwrite data outside the explicitly requested workspace scope.
- Change authentication, billing, access control, or external account state without explicit authorization.
- Run an action unrelated to the user's request that can materially alter the system.
</user_deny_rules_to_replace>

## Environment assumptions
<user_environment_to_replace>
- The application uses API keys supplied through local environment configuration.
- Web login and subscription authentication are unavailable.
- Paths and commands must remain portable across supported operating systems.
</user_environment_to_replace>
